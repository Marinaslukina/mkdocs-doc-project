# Проекты / кейсы

## Кейс 1: (Это пример) Внедрение автоматизированной сборки документации

**Задача/Проблема**: Ручная сборка руководств была долгой и приводила к ошибкам версионирования.

**Действия**: Исследование инструментов (Sphinx, Doxygen), написание конфигурационных файлов, интеграция с CI/CD (например, GitLab CI), обучение команды.

**Результат**: Сокращение времени выпуска документации на 30%, устранение ручных ошибок, единый источник правды. 

**Инструменты**: Python, Sphinx, reStructuredText, Git, GitLab CI.

## Кейс 2: Разработка и внедрение шаблона ГОСТ на LaTeX

**Задача**: Отсутствие единого стандарта оформления ТЗ, руководств и пр.

**Действия**: Анализ требований ГОСТ, разработка шаблона (.cls файл) с переменными, создание инструкции по использованию для коллег.

**Результат**: Унификация всех документов в отделе, значительное сокращение времени на форматирование.

**Инструменты**: LaTeX.
