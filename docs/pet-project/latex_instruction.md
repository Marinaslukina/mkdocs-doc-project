# Инструкция по работе с LaTeX

Данная инструкция представляет собой пошаговую инструкцию по работе с LaTeX-шаблоном. Цель документа — стандартизация оформления текстовых материалов и максимальное упрощение процесса их создания для всех сотрудников, независимо от уровня владения LaTeX.

В инструкции рассмотрены: базовая структура шаблона, порядок заполнения стандартных блоков (титульная страница, содержание, разделы). 

Для начала работы требуются только базовые навыки работы с текстовыми редакторами.

## Структура документа

1. Для каждого нового [проекта]("Отдельная папка, содержащая все исходные материалы и файлы, требуемые для генерации одного итогового документа") необходимо создать папку с наименованием проекта на английском языке, например, «OperatorManual». 

2. Далее в корневую папку вашего проекта необходимо скопировать предоставленный файл **main.tex**. Это главный файл, который управляет всем документом: в нём подключаются все остальные части (главы, стили оформления и прочее). 

3. В корневой папке следует создать подпапку **img**, в которой будут храниться все рисунки проекта.

4. В корневой папке следует создать подпапку **chapters**. В данной подпапке будут лежать все главы документа, титульный лист, лист регистрации изменений, перечень принятых сокращений, введение.

5. В подпапку **chapters** необходимо скопировать предоставленные файлы: 

    - **titel.tex** – для формирования титульного листа документа;
    - **abbreviations.tex** – для создания перечня сокращений;
    - **list.tex** – для создания листа регистрации изменений.

6. В корневой папке следует создать подпапку **variables**. В данную подпапку необходимо скопировать предоставленный файл **var.tex**. Данный файл содержит информацию для заполнения полей основного документа: децимальный номер изделия, код документа, вид документа, инвентарный номер, литера, год и т.д. Заполнение данного файла обязательно и будет указано ниже.

7. Файл **preample.tex** содержит стилевые настройки для документа и не должен изменяться разработчиком. Данный файл необходимо скопировать в основную папку проекта.

8. Файл **list.tex** содержит таблицу для создания листа регистрации изменений и не должен изменяться разработчиком.

## Подготовка файла с переменными данными

1. Перед началом разработки необходимо заполнить файл с переменными данными – **var.tex**, находящийся в подпапке **variables**.

2. Файл содержит следующий набор полей, значения которых следует указать:

    - наименование организации
```\newcommand{\CompanyName}{ООО «Рога и копыта»}```

    - наименование программы
```\newcommand{\ProductName}{Программный комплекс}```

    - децимальный номер программы
```\newcommand{\ProductDocNumber}{ИТСБ.12345-01}```

    - код документа
```\newcommand{\DocNumber}{34 01}```

    - вид документа
```\newcommand{\DocType}{Руководство оператора}```

    - при наличии части документа указать часть (с маленькой буквы!), при отсутствии – поставить пробел 
```\newcommand{\DocPart}{часть 1}```

    - год создания документа
```\newcommand{\DocDate}{2025}```

    - при наличии литеры указать ее, при отсутствии – поставить пробел
```\newcommand{\DocLiter}{О}```

    - при наличии инвентарного номера указать, при отсутствии – поставить пробел
```\newcommand{\DocInventoryNumber}{11111}```

3. Данные, указанные в файле с переменными данными, автоматически будут заполнены на титульном листе и колонтитулах. 

4. При необходимости использовать значения переменных данных в тексте документа следует вместо значения указать его поле. Таким образом, при изменении значения в файле **var.tex** оно изменится во всем документе.

## Сборка проекта

1. Сборка проекта осуществляется в файле **main.tex**.

2. В данном файле осуществляется подключение всех вспомогательных файлов, которые составляют документ, в том порядке, в котором они должны быть расположены в документе. 

3. Файл уже содержит все основные команды для работы. Однако, список, наименование и порядок вспомогательных файлов может быть изменен разработчиком. Поэтому ниже приведено описание команд файла. 

4. В файле необходимо найти следующие команды, отмечающие начало и конец основного содержания документа: 
   
```
\begin{document}  
<текст>
\end{document}```

Все, что находится между ними будет считаться частью основного текста документа.

Далее, в этом разделе, речь пойдет о командах, которые следует записывать между началом и концом документа.

5. Для подключения вспомогательного файла следует использовать следующую команду, где в фигурных скобках указывается путь до файла
```\include{chapters/titel}```
